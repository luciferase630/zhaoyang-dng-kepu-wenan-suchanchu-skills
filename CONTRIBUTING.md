# 维护者说明 / Maintainer Notes

## 中文版

### 开发校验

修改 skills、plugin metadata、MCP 配置或公开文档后，先同步跨 Agent skills 目录：

```bash
npm run sync:agent-skills
```

再运行：

```bash
npm run validate
npm pack --dry-run
```

可选的 OpenAI skill 校验：

```bash
PYTHONUTF8=1 python <path-to-skill-creator>/scripts/quick_validate.py plugins/zhaoyang-dng-kepu-wenan-suchanchu/skills/zhaoyang-explainer-scriptwriter
PYTHONUTF8=1 python <path-to-skill-creator>/scripts/quick_validate.py plugins/zhaoyang-dng-kepu-wenan-suchanchu/skills/zhaoyang-excalidraw-explainer
PYTHONUTF8=1 python <path-to-skill-creator>/scripts/quick_validate.py plugins/zhaoyang-dng-kepu-wenan-suchanchu/skills/zhaoyang-excalidraw-toolkit
```

### 发布

创建空 GitHub 仓库后推送本地仓库：

```bash
git remote add origin <repo-url>
git push -u origin main
```

如果远程地址已经存在，可以更新后再推送：

```bash
git remote set-url origin <repo-url>
git push
```

### 维护规则

- `plugins/zhaoyang-dng-kepu-wenan-suchanchu/skills` 是 canonical skills 目录。
- `.claude/skills` 和 `.agents/skills` 是镜像目录，必须由 `npm run sync:agent-skills` 生成。
- 默认 MCP 路径应保持 Node/npm 方案；Docker 只保留为 fallback 模板。
- 公开文档不要写入本机路径、账号邮箱、终端状态或素材来源目录。

## English Version

### Development Checks

After editing skills, plugin metadata, MCP configuration, or public docs, sync the cross-agent skill mirrors first:

```bash
npm run sync:agent-skills
```

Then run:

```bash
npm run validate
npm pack --dry-run
```

Optional OpenAI skill validation:

```bash
PYTHONUTF8=1 python <path-to-skill-creator>/scripts/quick_validate.py plugins/zhaoyang-dng-kepu-wenan-suchanchu/skills/zhaoyang-explainer-scriptwriter
PYTHONUTF8=1 python <path-to-skill-creator>/scripts/quick_validate.py plugins/zhaoyang-dng-kepu-wenan-suchanchu/skills/zhaoyang-excalidraw-explainer
PYTHONUTF8=1 python <path-to-skill-creator>/scripts/quick_validate.py plugins/zhaoyang-dng-kepu-wenan-suchanchu/skills/zhaoyang-excalidraw-toolkit
```

### Publishing

Create an empty GitHub repository, then push the local repository:

```bash
git remote add origin <repo-url>
git push -u origin main
```

For existing repositories, update the remote URL if needed:

```bash
git remote set-url origin <repo-url>
git push
```

### Maintenance Rules

- `plugins/zhaoyang-dng-kepu-wenan-suchanchu/skills` is the canonical skills directory.
- `.claude/skills` and `.agents/skills` are mirrors generated by `npm run sync:agent-skills`.
- The default MCP path should stay on Node/npm; Docker remains a fallback template only.
- Public docs must not include local machine paths, account emails, terminal state, or source-material directories.
